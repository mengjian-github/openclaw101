## 连接 Gmail 📧

这是你的第一个「实用技能」，也是大多数人最需要的。

### Step 1：创建 Google Cloud 项目

1. 打开 [console.cloud.google.com](https://console.cloud.google.com)
2. 创建一个新项目（名字随意，比如"My AI Assistant"）
3. 进入 **API 和服务 → 库**，搜索并启用：
   - Gmail API
   - Google Calendar API

### Step 2：创建 OAuth 凭证

1. 进入 **API 和服务 → 凭证**
2. 点击 **创建凭证 → OAuth 客户端 ID**
3. 应用类型选 **桌面应用**
4. 下载 JSON 文件，命名为 `credentials.json`
5. 放到你的工作目录（可以直接发给 AI，让它帮你放好就行）：`~/clawd/credentials.json`

### Step 3：安装 Gmail 技能

OpenClaw 的技能安装很简单（可以让 AI 帮你安装）：

```bash
clawdhub install gog
```

> 💡 **gog** 是 Google Workspace 技能，包含 Gmail + Google Calendar + Google Drive。

第一次运行时，它会打开一个浏览器链接让你授权 Google 账号。授权后，会生成一个 `token.json`——这就是你的钥匙。

### Step 4：测试

在 Telegram 里对助手说：

> 帮我看看 Gmail 今天有什么新邮件

如果一切正常，它会返回类似这样的消息：

> 📧 今天共 5 封新邮件：
> 1. **[重要]** 合作方回复 — 关于下周会议时间确认，需要你回复
> 2. GitHub — Your repository has been starred
> 3. Hetzner — Invoice for July
> 4. Newsletter — This Week in AI
> 5. 广告 — 已自动忽略

**注意到了吗？** 它不只是列出邮件，还帮你判断了优先级，标出了需要你处理的那封。这就是 AI 助手和普通邮件客户端的区别。

## 连接 Google Calendar 📅

有了 Gmail 的基础，日历就简单了——它们共享同一套 Google OAuth 认证。

因为你在安装 gog 技能时已经授权了 Google 账号，而且我们在 Step 1 启用了 Calendar API，日历功能直接就能用，不需要额外操作。

测试一下：

> 明天有什么安排？

> 📅 明天（7 月 19 日 周六）的安排：
> - 10:00-11:00 产品讨论会（腾讯会议）
> - 14:30 牙医预约（提醒：别忘了带医保卡）
> - 无其他安排，下午可以 deep work

更厉害的用法：

> 帮我在下周三下午 3 点创建一个会议，主题是"SEO 策略讨论"，时长 1 小时

> ✅ 已创建日历事件：
> - 📅 7 月 23 日（周三）15:00-16:00
> - 📝 SEO 策略讨论
> - 需要添加参会者吗？

它甚至能做冲突检测——如果你要约的时间段已经有事了，它会提醒你：

> ⚠️ **注意**：下周三 15:00-16:00 你已经有"客户电话"了。要我改成 16:30 开始吗？

## 连接搜索引擎 🔍

让助手能联网搜索，是打破「信息孤岛」的关键。

OpenClaw 支持多种搜索方式。最简单的是 Brave Search API：

### 配置 Brave Search

1. 去 [brave.com/search/api](https://brave.com/search/api) 注册免费账号
2. 获取 API Key
3. 在 OpenClaw 配置中添加：

```bash
openclaw configure --section web
```

向导会提示你输入 Brave Search API Key，自动写入配置。

配置好之后：

> 搜索一下 "OpenClaw alternatives 2026"

> 🔍 搜索结果摘要：
> OpenClaw 的主要替代品包括：
> 1. **AgentGPT** — 浏览器端运行，无需部署
> 2. **AutoGPT** — 老牌 Agent 项目，社区庞大
> 3. **CrewAI** — 多 Agent 协作框架
>
> 但这些都侧重"自主执行任务"，而 OpenClaw 更侧重"个人助手"——24/7 在线、聊天交互、技能系统。定位不同，不是直接竞品。

**它不是把搜索结果丢给你，而是帮你读完、总结、给出判断。** 这是搜索引擎做不到的事。

## 连接浏览器 🌐

有些信息搜索引擎找不到，比如一个特定网页的具体内容、一个需要登录的后台数据、一个动态加载的页面。

这时候你需要浏览器技能——让助手能「看到」网页。

OpenClaw 内置了浏览器技能（基于 Playwright），安装时已经自动配置好了。它可以：

- **访问任意 URL** 并提取内容
- **截图** 保存当前页面
- **交互操作** 点击、输入、滚动

用法示例：

> 帮我打开 https://kirkify.net 看看首页现在长什么样

> 🌐 已访问 kirkify.net：
> - 首页标题："Kirkify — Free Online Tools"
> - 主要工具列表：Name Generator，QR Code，Color Palette...
> - 页面加载正常，无明显错误
> [截图已保存]

更实用的场景：

> 帮我看看竞品 xyz.com 的定价页面

它会打开页面、提取定价信息、甚至和你之前看过的版本做对比。

> 🐱 **小墨碎碎念**：浏览器是我最喜欢的技能之一。以前孟健让我查竞品，我只能搜索。现在我可以直接打开对方网站，像一个真人用户一样浏览。有一次我发现竞品悄悄改了定价策略，第一时间通知了孟健。他说这一个发现就值一个月的服务器费用。

## 安全第一 🔐

接入了邮件、日历、浏览器——你的助手现在能触碰非常多的个人数据。安全是必须认真对待的事。

建议你先做一次安全体检（尤其是改过配置或暴露过网络端口之后）：

```bash
openclaw security audit
openclaw security audit --deep
openclaw security audit --fix  # 确认无误再用
```

> ⚠️ **浏览器相关提醒**：Browser Relay / CDP 基本等同"远程操作权限"，只建议在 localhost 或 tailnet 内使用，避免暴露到公网。

### 1. API Key 安全

- 永远不要把 API Key 提交到 Git
- 用环境变量或 `.env` 文件存储
- 定期轮换密钥

### 2. OAuth Token 安全

- `token.json` 等文件包含你的 Google 授权信息
- 确保文件权限设置正确：`chmod 600 token.json`
- 不要传到任何公开的地方

### 3. 权限最小化原则

只给助手它需要的权限。比如 Gmail，如果你只需要读邮件，就不要给"发送邮件"的权限。虽然 OpenClaw 默认需要确认才会发，但少一个权限就少一个风险。

### 4. 网络安全

- 服务器开启防火墙，只暴露必要端口
- SSH 用密钥认证，禁用密码登录
- 定期更新系统：`sudo apt update && sudo apt upgrade`

### 5. 行为边界

在 SOUL.md 和 AGENTS.md 中明确写好：
- 什么操作需要确认
- 什么数据不能外传
- 什么情况下该拒绝执行

> 💡 **安全不是装上锁就完事的，它是一种习惯**：养成良好的安全习惯：API Key 不入库、Token 文件设权限、权限最小化、行为边界写清楚。

> 🐱 **小墨碎碎念**：孟健有一次让我查一个同事的邮件记录，我拒绝了。因为 SOUL.md 里写了"只处理主人自己的数据"。他当时有点惊讶，然后说"好，这条规则留着"。边界清晰，信任才稳固。
